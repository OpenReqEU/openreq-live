A "Smart House" is a home management system that allows home residents to easily manage their daily lives by providing for a lifestyle that brings together security, environmental and energy management (temperature, humidity and lighting), entertainment, and communications.

Requirement dependencies can ONLY be identified correctly, after you completely understood the full content of the given PDF document.

On the next page you find a list of requirement pairs which has to be analyzed by you. Your task is to manually analyze the two requirements of each requirement pair individually and then to classify whether there exists a dependency between the two different requirements or not. The analyzation/classification of each pair should be done based on the descriptions of both requirements and the background knowledge you gained when reading the PDF document.

Dependency types:

For every classified dependency you also have to specify the type of the dependency. Thereby, you have to choose one of the following 4 different dependency types:

  • REQUIRES dependency:
    x REQUIRES y: For the implementation of the Requirement x, Requirement y needs to be implemented first. Attention: A REQUIRES DEPENDENCY is always unidirectional, that means (x REQUIRES y) is NOT the same as (y REQUIRES x).
  • EXCLUDES dependency:
    x EXCLUDES y: Requirement x and requirement y can not implemented in the same release, which means that the implementation of x forbids the implementation of y and vice versa.
  • IS SIMILAR TO dependency:
    Rx IS SIMILAR TO Ry: Both Requirements are similar or related to each other based on their descriptions.
  • IS PART OF dependency:
    x IS PART OF y: Requirement x is a part of requirement y means that y includes x. This can also be interpreted as follow: x is child/subrequirement of y, whereby y can have several children/subrequirements.

Examples for dependency types:

To understand the meaning of the 4 given dependency types a demonstrative example is provided. Consider the following subset of requirements for the development of a sportswatch:
Requirement a: For statistical purposes, a distance measurement is necessary.
Requirement b: The average length of steps are required.
Requirement c: Number of steps have to be measured with a pedometer.
Requirement d: The sportswatch should only be based on iOS.
Requirement e: The sportswatch should only be based on Android.

Dependencies of unidirectional type REQUIRES:
Requirement a REQUIRES requirement b
Requirement a REQUIRES requirement c

Reason: in order to implement requirement a requirement b must be implemented first because for the calculation of the distance the average length of steps is needed (i.e., a REQUIRES b). Also, the number of steps must be measured as well to compute the distance (i.e., a REQUIRES c).

Dependencies of unidirectional type IS PART OF:
Requirement c IS PART OF requirement a
Requirement b IS PART OF requirement a

Dependencies of bidirectional type EXCLUDES:
Requirement d EXCLUDES requirement e (is equivalent to e EXCLUDES d, because it's a bidirectional dependency)

Dependencies of bidirectional type IS SIMILAR TO:
Requirement b IS SIMILAR TO requirement a (is equivalent to a IS SIMILAR TO b, because it's a bidirectional dependency)
